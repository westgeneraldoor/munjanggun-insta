> 📅 작성일: 2026-05-13

## 1. 핵심 비전

**"문장군의 인스타 콘텐츠 생산을 시스템화하여, AppSheet 데이터와 블로그 자산을 기반으로 다양한 포맷의 인스타 콘텐츠를 대량 생산하는 파이프라인"**

블로그엔진이 블로그 글을 체계적으로 찍어내듯,
인스타엔진은 숏폼·캐러셀 소재를 체계적으로 찍어낸다.

---

## 2. 문제 정의

### 현재 상황
- 문장군 블로그 파이프라인은 가동 중 (일 4~7편 발행)
- 인스타그램은 방치 상태. 테스트 수준의 숏폼 2개만 광고 중
- TTS 대화형 숏폼을 테스트했고, **CPC 150원, CTR 10%+, 재생율 45%+** 달성 (검증됨)
- 하지만 제작 과정이 수동이고 도구 6개를 넘나들어 양산 불가능
- 인스타 계정 팔로워 ~1,000명 (대부분 유령 → 새 계정으로 전환 결정)

### 핵심 문제
1. **제작 시간이 너무 오래 걸린다** → 시스템화 필요
2. **비포/애프터만으로는 독창성이 없다** → 데이터 기반 차별화 필요
3. **릴스가 주력이지만 직접 촬영 여력 없음** → 캐러셀+음악으로 릴스 알고리즘 활용

## 3. 타겟 사용자 & 콘텐츠 소비자

### 운영자 (문장군 마케팅 담당 — 바이브코더)
- 블로그엔진으로 글을 쓰고, 같은 리듬으로 인스타 콘텐츠도 뽑고 싶음
- 구글 나노바나나 + 플로우 기반 영상 제작 경험 있음
- AI 도구 활용 능숙. 시스템만 갖추면 공격적 양산 가능

### 콘텐츠 소비자 (인스타 시청자)
- 30~50대, 여성 주력 (남성도 포함)
- 아파트 인테리어/리모델링 관심
- "우리 집도 이렇게 되나?" → DM 공유
- "나중에 시공할 때 참고" → 📌 저장

## 4. 문장군만의 독보적 자산

> ⭐ 다른 인테리어 업체가 절대 못 따라하는 것

| 자산 | 상세 | 콘텐츠 활용 |
|------|------|-----------|
| **AppSheet 전수 데이터** | 이름, 주소, 실측접수일/완료일/시공일, 견적금액, 견적상세, 비포사진, 애프터사진, 시공유형, 종류, 스펙, 현장특이사항 | 데이터 인사이트 콘텐츠, 스토리 소재 발굴 |
| **하루 최대 35현장** | 매일 수십 건의 시공 데이터가 자동 축적 | 콘텐츠 소재가 매일 자동 생산 |
| **15,000 리뷰** | 3연동중문 리뷰 15,000개, 플레이스 4,000개 | 사회적 증거(Social Proof) |
| **TTS 대화형 포맷 (검증됨)** | CPC 150원, CTR 10%+, 재생율 45%+ | 킬러 포맷 확장 |
| **구글 나노+플로우 영상 제작** | 이미지→영상 변환 경험 보유 | 숏폼 양산 가능 |
| **블로그 콘텐츠 파이프라인** | 일 4~7편 블로그 발행 중 | 블로그→캐러셀 파생 |

## 5. 인스타 콘텐츠 3축 전략

### 축 1: TTS 대화형 숏폼 (50%) — 공유 킬러

**콘셉트:** 고객(여)과 문장군(남)이 대화하는 30초 이하 숏폼

**소재 유형:**
- AppSheet 실제 현장 데이터 기반 에피소드 ("20년 된 아파트인데 중문 달 수 있어요?")
- 블로그 글의 핵심 메시지를 대화체로 변환
- "현장특이사항" 필드에서 흥미로운 에피소드 발굴
- 고객 페인포인트 재연 → 해결 과정 → 결과

**현재 제작 프로세스 (수동, 느림):**
```
AI 스크립트 → 제미나이 TTS → 씬 프롬프트 추출 → 구글 플로우+나노바나나 이미지
→ 이미지→영상 변환 → 자막+편집 → 업로드
```

**인스타엔진이 자동화할 부분:**
```
✅ 스크립트 생성 (대화 대본 — 고객/문장군 역할 구분)
✅ TTS 대본 구조화 (제미나이에 바로 넣을 수 있는 형태)
✅ 씬별 이미지 생성 프롬프트 (구글 플로우/나노바나나용)
✅ 씬별 영상 변환 프롬프트 (구글 플로우용)
✅ 자막 텍스트 (타이밍 포함)
✅ 인스타 캡션 + 해시태그 + 음악 추천
```

**사용자가 수동으로 할 부분:**
```
🔧 구글 플로우에서 프롬프트 입력 → 이미지/영상 생성
🔧 캡컷 등에서 편집 (소재는 다 나와있으니 조립만)
🔧 인스타 업로드
```

**왜 먹히는가:**
- DM 공유 (친구에게 보내기) = 2026 알고리즘 최강 시그널
- 대화 구조 → 끝까지 시청 (재생율 45% 검증)
- 감정 이입 → "나도 이런 고민인데!" → 공유

### 축 2: 데이터 인사이트 캐러셀 (30%) — 저장 킬러

**콘셉트:** AppSheet 실데이터 기반 "~TOP 3", "~공개" 캐러셀 + 음악

**소재 유형:**
- "이번 달 3연동중문 시공 150건 — 가장 인기 색상 TOP 3" (실수치)
- "30평 vs 40평, 인기 조합이 다른 이유" (데이터 비교)
- "시공팀이 말하는 실패 없는 조합" (현장 경험)
- "지역별 인기 시공 트렌드: 서울 vs 경기" (지역 데이터)

**구조:** 5~8장 캐러셀 + 음악 (→ 릴스 피드 노출)
```
[1장] 훅 — "이번 달 시공 150건 데이터 공개"
[2~4장] 핵심 인사이트 (실수치 기반)
[5장] 브랜드 — "문장군은?" + 강점
[6장] CTA — "📌 저장하고 시공할 때 꺼내보세요"
```

**인스타엔진이 자동화할 부분:**
```
✅ AppSheet 데이터 요약 요청 (사용자에게 질문 or 데이터 입력받기)
✅ 인사이트 추출 → 슬라이드 텍스트 생성
✅ HTML 캐러셀 생성 (기존 템플릿 활용)
✅ 캡션 + 해시태그 + 음악 추천
```

**왜 먹히는가:**
- 구체적 숫자 = 신뢰 → 📌 저장
- "다른 사람들은 뭘 선택했지?" → 참고용 저장
- 음악 추가 → 릴스 피드 노출 → 신규 도달

### 축 3: 블로그 파생 캐러셀 (20%) — 신뢰 빌더

**콘셉트:** 블로그 글 핵심 내용을 캐러셀로 재가공

**소재 유형:**
- 블로그 정보글 → 핵심 포인트 나열형 캐러셀
- 블로그 비교글 → 비교표 캐러셀
- 블로그 가격가이드 → 비용 프레이밍 캐러셀

**인스타엔진이 자동화할 부분:**
```
✅ 블로그 글 파싱 → 핵심 메시지 추출
✅ 템플릿 선택 → 슬라이드 텍스트 생성
✅ HTML 캐러셀 생성
✅ 캡션 + 해시태그 + 음악 추천
```

## 6. 인스타 알고리즘 레퍼런스 (2026 리서치)

### 알고리즘 핵심 시그널

| 시그널 | 중요도 | 설명 |
|--------|--------|------|
| **DM 공유** | ★★★★★ | 친구에게 보내는 콘텐츠 = 최강 배포 신호 |
| **저장** | ★★★★★ | 나중에 다시 볼 콘텐츠 |
| **재생 완료율** | ★★★★ | 릴스/숏폼에서 끝까지 시청 |
| **체류시간** | ★★★★ | 캐러셀 스와이프 시간 |
| **댓글/좋아요** | ★★★ | 여전히 중요하지만 공유/저장보다 약함 |

### 캐러셀+음악 = 릴스 취급
- 음악을 추가한 캐러셀은 **릴스 피드에 노출** 가능
- 기존 팔로워 외 신규 오디언스에게 도달
- 캐러셀 "세컨드 찬스" — 안 본 슬라이드를 인스타가 재노출

### 콘텐츠 프레이밍 원칙
- ❌ "문짝 교체했습니다" → ✅ "이 현관이 이렇게 바뀐다고?"
- ❌ "중문 설치" → ✅ "10년 된 아파트가 신축처럼"
- 변신/변화/결과 중심 프레이밍 필수

### 콘텐츠 비율 공식 (리서치 기반)
- 가치 제공 (정보/교육/엔터테인먼트): **75%**
- 브랜드 홍보/판매: **25%**
- → 광고 느낌 아닌 "도움이 되는 전문가 채널" 포지셔닝

## 7. 계정 전략

### 새 계정 시작 (확정)
- 기존 계정 팔로워 ~1,000명 (대부분 유령 — 사기 업체 유입)
- 기존 계정 히스토리 짧고 지킬 자산 없음
- 새 계정으로 처음부터 "인테리어/시공 전문" 알고리즘 학습 유도

### 프로필 최적화 (PRD에서 상세화)
- 바이오: 핵심 키워드 + CTA (네이버 예약 링크)
- 하이라이트: 시공 후기 / 시공 과정 / Q&A / 견적 안내
- 프로필 사진: 문장군 로고 또는 브랜드 아이덴티티

### 발행 전략
- 목표: 주 5~7개 (일 1개 이상)
- 비율: 축1 숏폼 50% + 축2 데이터 캐러셀 30% + 축3 블로그 파생 20%
- 초기 30개 콘텐츠 시딩 전략 필요 (PRD에서 구체화)

## 8. 기존 자산 활용

### 인스타그램 카드뉴스 자동생성기 (기존 프로젝트)
| 자산 | 위치 | 활용 방안 |
|------|------|----------|
| HTML 템플릿 3종 | `templates/type-a,b,c.html` | 축2, 축3 캐러셀용 재사용 + 정보형 추가 |
| 디자인 시스템 CSS | `styles/base.css` | 그대로 재사용 |
| Puppeteer 캡처 | `capture.js` | HTML→PNG 변환에 활용 |
| 브랜드 컬러/로고 | CSS + 이미지 | 일관된 브랜드 적용 |

### 블로그 파이프라인 (문장군블로그)
- posts/ 폴더의 블로그 글 → 축3 파생 소재
- BRAND_CONTEXT.md → 톤/CTA/금지어 공유
- SEO_KEYWORD_RESEARCH.md → 해시태그 참조
