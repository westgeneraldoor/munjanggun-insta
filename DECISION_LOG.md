# 문장군 인스타그램 결정 로그

> 목적: 채팅방이 바뀌어도 중요한 결정의 이유와 영향 범위를 잃지 않기 위한 장기 기록.
> `_context.md`는 현재 상태용, 이 파일은 누적 결정 보관용이다.

## 문서 우선순위

충돌이 생기면 아래 순서로 판단한다.

1. `AGENTS.md` — 프로젝트 헌법, 브랜드/금지사항/실행 트리거
2. `DECISION_LOG.md` — 장기 결정과 변경 이유
3. `PRD_*.md` — 제품·기능 요구사항
4. `BRAND_CONTEXT.md`, `INSTAGRAM_OPERATING_GUIDE.md`, `INSTAGRAM_CONTENT_STRATEGY.md` — 브랜드·운영·전략 기준
5. 전역 `SKILL.md` — 실행 방법론 (인스타엔진)
6. `_context.md` — 현재 작업 상태
7. 채팅 내용 — 즉시 맥락. 장기 결정은 반드시 파일로 승격

## DEC-019: 프로젝트 분리 — 인스타그램 독립 프로젝트로 전환 — 2026-05-18
- **배경:** 블로그 프로젝트에서 인스타그램 시스템이 함께 성장하면서 도메인 경계가 무너짐. 에이전트 역할 불명확, 컨텍스트 비대, 규칙 충돌 발생.

- **영향:** 전체 프로젝트 구조 재편. 새 AGENTS.md, GEMINI.md, DECISION_LOG.md, _context.md 작성.

## DEC-020: 004~006 테마는 사용자의 명시적 지시에 따라 캐러셀 단독 제작으로 예외 인정 — 2026-05-20

- **결정:** 004, 005, 006 테마(기존 005~007 폴더)는 숏폼 없이 캐러셀 단독 포맷으로 최종 확정 및 승인 처리하며 예외를 인정함.

- **재검토 조건:** 사용자가 향후 필요에 의해 해당 테마들의 숏폼 제작을 추가로 지시할 때.

## DEC-021: 총괄매니저 오더 수립 및 하달 대기 규칙 — 2026-05-20
- **배경:** 총괄 매니저가 선제적으로 작업지시서를 갱신하거나 서브에이전트에 지시를 내리면 사용자의 통제 범위를 벗어날 수 있음.
- **결정:** 총괄 매니저는 사용자의 명시적인 실행 오더(예: "Theme 8 시작해줘", "진행해")를 받기 전까지는 `_order.md`를 신규 작성하거나 서브에이전트에게 지시를 하달하지 않는다. 작업 완료 보고 후 다음 단계를 안내하고 즉시 **대기(STOP)**한다.
- **영향:** 총괄매니저 실행 프로세스, 서브에이전트 협업 파이프라인.
- **재검토 조건:** 사용자가 명시적으로 완전 자동화 또는 연속 태스크 처리를 요구할 때.
